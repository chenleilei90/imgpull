# 01 产品需求

## 产品定位

ImgPull 是面向 DevOps、云原生工程师、运维和开发团队的镜像同步平台。用户提交 Docker Hub、GHCR、Quay 等公开镜像地址，平台将镜像同步到用户自己的阿里云 ACR、腾讯云 TCR、华为云 SWR、火山云 CR、自建 Harbor 或通用 Docker Registry。

当前版本是 **P0 前端演示版**，用于页面演示和产品流程评审，不能真实拉取或推送镜像。

## 目标用户

- 国内开发者：需要稳定获取海外公开容器镜像。
- 运维人员：需要把外部镜像同步到企业私有仓库。
- DevOps 团队：需要可追踪、可审计、可复用的镜像同步流程。
- 小团队或企业：需要积分、充值、任务状态、失败返还和后台管理能力。

## P0 核心闭环

1. 用户注册或登录。
2. 用户获得积分：注册送、活动赠送、管理员人工充值或会员权益。
3. 用户配置自己的私有仓库凭据和项目 / namespace。
4. 用户创建单个镜像任务，或使用轻量批量导入。
5. 系统解析源镜像和目标项目，预估积分。
6. 系统按独立任务冻结积分。
7. Worker 后续按单任务执行镜像复制。
8. 成功后结算积分，失败后按单任务返还冻结积分。
9. 用户查看任务状态、日志、错误码、tag pull 和 digest pull 命令。

## 单任务、多目标和批量导入模型

P0 支持：

- 单个源镜像同步到一个目标项目。
- 单个源镜像同步到多个目标项目。
- 多个源镜像批量导入，并同步到一个或多个目标项目。

核心规则：

```text
1 个源镜像 + 1 个目标仓库项目 = 1 条独立 ImageTask
```

示例：

```text
source: docker.io/library/nginx:latest
targets:
- 阿里云 ACR / ops
- 阿里云 ACR / platform
- Harbor / devops
```

最终创建 3 条独立任务，每条任务都有独立的 taskNo、积分冻结、Worker 分配、日志、成功、失败和返还。

P0 上限：

- 源镜像最多 50 行。
- 目标项目最多 3 个。
- 单次最多创建 150 条 ImageTask。

超过上限时，前端禁用提交并提示用户减少源镜像或目标项目数量。

## 任务编号

为了方便用户、客服和管理员沟通，P0 增加用户可见任务编号：

```text
IMG-ABC-1234
```

批量导入批次编号：

```text
BAT-ABC-1234
```

说明：

- taskNo 是业务展示编号，不替代数据库内部 id。
- taskNo 必须唯一。
- 搜索支持大小写不敏感、横线和空格不敏感。
- 用户按 taskNo 查询任务时仍必须校验任务归属。
- 管理员可以全局搜索 taskNo。

## 目标项目 / namespace

P0 不自动读取云厂商 namespace / project。

原因：

- 通用 Docker Registry V2 没有统一的项目列表接口。
- 阿里云、腾讯云、华为云、火山云需要各自 OpenAPI。
- P0 不要求用户提供云厂商 AccessKey。

P0 做法：

- 用户在私有仓库管理中手动维护项目 / namespace 列表。
- 新建任务时，从已维护的项目列表中搜索和多选。
- 如果项目不存在，提示用户先去云厂商或 Harbor 控制台创建。
- 目标项目状态和 push 权限仍需要在任务创建或测试连接时校验。

P1 规划：

- Harbor project 自动读取。
- 阿里云 ACR namespace 自动读取。
- 腾讯云 TCR namespace 自动读取。
- 华为云 SWR namespace / organization 自动读取。
- 火山云 CR namespace 自动读取。

## P0 不做

- 不接真实后端 API。
- 不接真实数据库。
- 不接真实 Worker。
- 不执行真实 registry copy。
- 不接真实支付宝 / 微信支付。
- 不接云厂商 OpenAPI。
- 不自动创建 namespace / project。
- 不做 CSV / Excel 上传。
- 不做 docker-compose / Kubernetes YAML 解析。
- 不做批量模板和定时同步。
- 不把 taskNo 当作访问凭证。

## P1 候选

- Harbor / 云厂商项目自动读取。
- CSV / Excel 导入。
- docker-compose / Kubernetes YAML 解析。
- 批量模板。
- 定时同步。
- 团队空间和成员权限。
- API / CLI。
- 真实 Worker 镜像和二进制分发。
