ImgPull

海外镜像统一获取，自动分发到你的专属国内地址

用户在首页搜索镜像或输入串码进入控制台。系统优先命中本地 Harbor 默认缓存仓库，未命中时再回退到 Docker Hub 搜索，并把镜像同步到默认缓存仓库后分发到你的专属命名空间。

已有串码
输入串码后可直接进入用户控制台。
购买串码

购买后立即获得授权串码

填写邮箱后可生成新的授权串码。当前版本优先在页面直接展示和复制，后续接入 SMTP 后也可同步发到邮箱。

购买后会立即返回串码，可直接复制使用。
购买成功
系统已生成新的授权串码：
-
用户控制台 交付镜像页 后台管理
-镜像目录总数
-本地缓存镜像
-有效用户数
-排队任务数
搜索逻辑

优先本地缓存，再查 Docker Hub

平台默认优先搜索本地 Harbor 默认缓存仓库。只有当本地没有命中或用户主动选择综合搜索时，系统才会展示 Docker Hub 结果。

获取流程

默认仓库缓存，用户只拿专属地址

镜像先进入默认缓存仓库，再分发到用户自己的 Harbor 命名空间。普通用户不直接操作默认仓库，只复制自己的专属拉取地址。

保留策略

缓存可续期，用户支持软删除

后台可设置缓存保留天数，镜像被再次请求时会自动刷新缓存期限。用户删除后先进入 30 天保留期，再决定是否彻底清理。

当前版本说明

支持安装向导、SQLite / MySQL 选择、Harbor 可选配置、购买串码、用户控制台和后台设置。
当前运行时主逻辑仍以 SQLite 兼容路径为主，MySQL 主要完成了安装配置和连接测试。
真实 Harbor API 联动和完整的 pull / tag / push worker 仍在后续完善中。
